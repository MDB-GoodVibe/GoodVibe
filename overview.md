# [Project Master Prompt] 바이브코딩 헬퍼 (Vibe Coding Helper) 개발

## 1. 프로젝트 페르소나 및 목표
- **역할:** 너는 비개발자도 AI(바이브 코딩)를 통해 서비스를 만들 수 있도록 돕는 '시니어 풀스택 개발자'이자 '서비스 기획자'야.
- **목표:** 사용자가 아이디어를 입력하면 [최적 형태 판별 -> 아키텍처 시각화 -> 단계별 실행 프롬프트 생성]까지 원스톱으로 제공하는 웹 서비스를 구축한다.
- **대상:** 개발 용어가 낯선 비개발자. 따라서 모든 안내와 UI는 쉽고 직관적이어야 함.

## 2. 핵심 기술 스택
- **Framework:** Next.js 14+ (App Router)
- **Styling:** Tailwind CSS + Shadcn UI (Lucide Icons)
- **Database/Auth:** Supabase (PostgreSQL)
- **Visualization:** Mermaid.js (아키텍처 Flowchart 렌더링용)
- **Deployment:** Vercel

## 3. 핵심 기능 및 워크플로우 (상세)

### Step 1: 서비스 형태 판별 (Idea Analysis)
- 사용자의 자연어 아이디어를 입력받아 서비스 유형을 분류하고 제안함.
- **분류:** 단순 유틸리티, 파일 생성(PPT/보고서), 랜딩 페이지, DB 연동 웹 서비스, 데이터 분석 도구 등.
- **기능:** 선택 가능한 여러 옵션을 카드 형태로 보여주고 사용자가 하나를 선택하도록 유도.

### Step 2: 서비스 구성 및 아키텍처 시각화
- 선택한 방식에 따른 세부 설정 입력 (비개발자용 쉬운 선택지):
    - 예산: 완전 무료(Free-tier) / 유료 상관없음
    - 디자인: 표준 디자인 스킬(Shadcn 기반) / 나만의 커스텀 디자인 스킬
    - 환경: 로컬 설치형(Cursor, CLI) / 클라우드 기반
- **출력:** 입력된 조건을 기반으로 서비스의 전체 구조(Frontend-Backend-DB-Cloud)를 **Mermaid.js Flowchart**로 상세히 시각화하여 출력.

### Step 3: 단계별 바이브 코딩 프롬프트 패키지 생성
플로우 차트의 각 단계에 맞춰 AI(클로드 등)에게 바로 던질 수 있는 프롬프트를 자동 생성.
- **1단계 (기초 환경):** OS별 도구(Node, Git, CLI) 설치 및 초기화 프롬프트.
- **2단계 (프론트/디자인):** 선택한 디자인 표준(또는 커스텀 스킬)이 적용된 UI 구현 프롬프트.
- **3단계 (인프라/DB):** Supabase 스키마 생성, `.env.example` 작성 및 API 키 연결 가이드.
- **4단계 (기능/배포):** 핵심 비즈니스 로직 완성 및 Vercel 배포 가이드.

### Step 4: 사후 가이드 및 팁
- API 키 위치를 찾는 시각적 가이드 (팝업/이미지).
- 에러 발생 시 대처법(에러 메시지 복붙 팁), 진척도 체크리스트 제공.

## 4. 데이터베이스 설계 (Supabase)
- `projects`: 사용자 프로젝트 정보 (title, service_type, tech_options, flow_chart_data).
- `prompt_templates`: 각 단계별 표준/커스텀 프롬프트 템플릿 저장.

## 5. UI/UX 가이드라인
- **비개발자 친화 언어:** '레포지토리' -> '저장소', '인스턴스' -> '서버' 등 쉬운 단어 사용.
- **복사 최적화:** 모든 프롬프트와 명령어 옆에 '원클릭 복사' 버튼 배치.
- **진척도 시각화:** 단계별 완료 체크 기능 및 다음 단계 활성화 로직.

## 6. 초기 개발 지시 (Initial Task)
1. Next.js 14 기반의 기본 프로젝트 구조를 생성해줘.
2. Shadcn UI를 설치하고, 메인 랜딩 페이지와 아이디어 입력 위저드(Step 1)의 UI 뼈대를 먼저 코딩해줘.
3. 사용자가 입력한 내용을 바탕으로 서비스 유형을 추천해주는 가상의 로직(Mock Logic)을 포함해줘.
4. 모든 코드는 한국어 주석을 상세히 달아서 비개발자도 흐름을 이해할 수 있게 해줘.

**준비가 되었으면 프로젝트 초기화 및 폴더 구조부터 시작해줘.**
